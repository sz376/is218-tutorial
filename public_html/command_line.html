How to host a website on Azure Cloud

Command Line/Linux Commands

Command Line

  • ls - list files
  • mkdir - make directory
  • cd (tab to auto complete) - change current directory
  • pwd - present working directory
  • touch -makes file
  • rm - remove file
  • -r - recursive for removing directories
  • mv - renames files (mv name_file new_file) OR move files (mv name_file directory/) must have slash for directory
  • cd .. - go back a directory
  • cd ~ - home directory
  • rm -rf .git - destroys git repo
  • vi - visual editor in terminal

Docker Commands

  • docker compose up -d (daemon) - creates and starts a container for a service
  • docker compose down - stops and removes containers and everything attached
  • docker ps - list all running docker containers
  • docker kill CONTAINER-ID - stops container
  • docker build -t {image name} . - build docker container image
  • docker login -u {docker username} - log into docker hub
  • docker tag {image name} {docker username}/{image name} - adds a tag to a docker image for pushing into docker cloud
  • docker push {docker username}/{docker repo name} - pushes an image into docker hub
  • docker login azure - login into azure cloud
  • docker context create aci {context_name} - create docker context
  • docker --context {context_name} - run commands with a different context while staying in the default context.
  • docker context use {context} - switch to different context

Git Commands

  • git init - creates local git repo
  • git status - checks if there are updated files/new files that need to be added
  • git add FILENAME - adds file to be committed
  • git commit -m “COMMIT MESSAGE” - commits changes to be pushed
  • git remote add origin LINK - make local repository push to remote repository
  • git push -u origin BRANCH - pushes commits to chosen branch
  • git merge master - merges changes from different branch
  • git pull origin BRANCH - does git fetch and git merge
